## はじめに

「卵を積んで、割って、料理を完成させる」落ちものパズル **タマリス** を Claude Code と一緒に作り、Cloudflare Pages でリリースしました。

https://tamaris.pages.dev/

この記事は、企画からリリース、そしてリリース翌日の改善までを AI と二人三脚で進めた開発記です。コードのほぼすべてを Claude Code が書き、私は企画・仕様の判断・実機プレイでのフィードバックに徹しました。

## ゲームの概要

- 上から落ちてくる卵を左右に動かして積む、いわゆる落ちものパズル
- 揃えて割ると卵料理が進行。ステージごとのノルマを達成すると「とろ〜りオムライス」などの料理が完成
- 特殊卵（温泉卵・ゆで卵・皮蛋・レインボー・アヒル）やハンマー、割りすぎゲージなどのギミック
- モードは3種類：ノルマモード / スコアモード / ゆで卵モード（時間経過で卵が硬質化）
- クリアを重ねるとレシピ図鑑に料理がコレクションされる

技術構成はシンプルです。

| 項目 | 採用技術 |
| --- | --- |
| ゲームエンジン | Phaser 3（プレーンJS、バンドラなし） |
| 素材 | 手書きSVG（卵10種・料理・UI） |
| BGM・効果音 | WebAudio でリアルタイム生成（音源ファイルなし） |
| ホスティング | Cloudflare Pages |
| 統計収集 | Pages Functions + D1 |

## Claude Codeとの分業

開発の役割分担は自然とこうなりました。

- **私**: 企画（卵料理×パズルという題材）、仕様の最終判断、実機（スマホ）でのプレイテスト、違和感の報告
- **Claude Code**: 実装全般、SVG素材の作成、WebAudioでの作曲、バグ調査、デプロイ、統計レポート

印象的だったのは、私の報告が曖昧でも会話で仕様が固まっていくことです。「横スワイプ後に指を離すと卵が勝手に落ちる」「NEXTの表示がはみ出してる」といった実機での違和感を伝えると、原因を特定して直してくれる。逆に私の要望が矛盾していると、選択肢を出して確認してくれる。この往復が対面ペアプロに近い感覚でした。

## 開発で面白かったところ

### BGMも効果音もコードで生成

音源ファイルを一切使わず、WebAudio で ♩=76 の Cmaj7→Am7→Fmaj7→G ループのゆるいBGMを生成しています。落下時のホイッスル、着地の「ドスッ」という低域の効いた音もすべてコードです。「ゆるい雰囲気で」と伝えたらコード進行から提案してくれました。

### ゲームオーバーは理由別に演出

殻の詰まり・割りすぎなど敗因ごとに見出し・色・シェフの怒号が変わります。敗因が伝わるとリトライの動機になる、という狙いです。

### 統計駆動でバランス調整

ゲームバランスは感覚ではなく統計で調整する方針にしました。Pages Functions + D1 で匿名イベント（clear / gameover / abandon / session / error）を収集し、`./stats.sh` で集計レポートが出ます。リリース後、実際に「ノルマがきつい」というデータと体感からステージノルマを 30/40/50 → 15/25/40 に緩和しました。

## リリース翌日の改善

リリースして自分でプレイして、すぐ気づいたことがありました。

**1回目のクリアで完成した料理がドーンと表示される。でも図鑑の解放はまだ先。これでは図鑑を開いたときのサプライズがない。**

Claude Code と仕様を往復して、こう変えました。

- 図鑑の解放は「ノルマモードの全クリア（最終ステージクリア）回数」ベースに。1周目→1品、2周目→2品、3周目→3品
- 未解放の料理はクリア画面でもシルエット表示＋「なにかが完成した…？」
- 代わりに、ノルマモードは続きのステージから再開できるように（他のモードで遊ぶとリセット）

この仕様、最初に私が伝えた案を Claude が実装し始めたところで「いや、そうじゃなくて」と2回ひっくり返しています。それでも会話で追従してくれるので、仕様を文書化してから渡すより速かったです。

検証も Claude Code がブラウザ自動化でローカル環境を操作し、「ステージ1周回では図鑑が貯まらない」「全クリアで解放演出が出る」「再開ポイントがスコア持ち越しで動く」まで確認してからデプロイしました。

## おまけ：名前の商標調査

リリース後に「カタカナで『タマリス』の化粧品ブランドがあるらしい」と気づいて冷や汗をかきました。Claude Code に J-PlatPat で称呼「タマリス」を検索してもらったところ：

- 化粧品のタマリス（ヘアケアメーカー）は第3類中心で、ゲームとは非類似の区分
- 唯一ゲームに掛かり得るのは、ある信用金庫のマスコット「たまりす」（第9類に電子計算機用プログラムを含む図形商標）

無料・非商用の範囲では問題になる可能性が低い、有料化や商標登録を考えるときは弁理士に相談、という結論まで出して調査メモをリポジトリに残してくれました。名前を決めたら早めに調べるべきでした。

## まとめ

- 企画と判断に集中し、実装をAIに任せる開発は個人ゲーム開発と相性がいい
- 実機の違和感を口頭レベルで報告→修正、の往復が高速
- 統計収集を最初から仕込んでおくと、リリース後の調整が「データを見て直す」になる
- 名前の商標調査は早めに
