# エージェントを“使い始めて”見えてきた課題”  
**― 1つ目のエージェントが動き出した後のリアル ―**

前回までで、ついに **最初のエージェントが誕生した**。  
あの瞬間は本当に嬉しかった。  
「おお、動いた…！」って、ひとりで小さくガッツポーズしたのを覚えている ✨

でも、ここからが本番だった。

---

## 🎉 1つ目のエージェントは“完成”じゃなくて“スタート”だった

最初のエージェントは、  
こちらの意図を読み取り、レビューを返してくれる。

「これは革命だ…！」と思った。

けれど、使い始めてすぐに気づく。

> **あれ、これだけじゃ業務は回らないぞ 🤔**

レビューはできる。  
でも、レビューするための材料は？  
TestRailへの反映は？  
日次の記録は？  
仕様変更の検知は？

**1体では、業務の一部分しか自動化できない。**

---

## 🎨 図解：1つ目のエージェント誕生後の“気づき”

```
┌──────────────────────────────┐
│   🐣 最初のエージェント誕生！ │
└───────────────┬──────────────┘
                │
                ▼
        「レビューはできるけど…？」
                │
                ▼
┌──────────────────────────────┐
│   🤔 これだけじゃ業務は回らない…   │
│   ・スコープは？                    │
│   ・TestRailは？                    │
│   ・日次記録は？                    │
│   ・仕様変更は？                    │
└──────────────────────────────┘
```

---

## 🧱 エージェントを使うことで見えてきた3つの壁

### ① エージェントは万能じゃない  
最初のエージェントは優秀だったけれど、  
万能ではなかった。

- スコープは作れない  
- TestRailには出力できない  
- 仕様変更には気づけない  

---

### ② 役割を詰め込みすぎると、すぐ破綻する

```
┌──────────────────────────────┐
│   🧠 1つのエージェントに全部任せる │
└───────────────┬──────────────┘
                │
                ▼
        ┌────────────────┐
        │  😵‍💫 出力がブレ始める │
        └────────────────┘
                │
                ▼
┌──────────────────────────────┐
│   ・文脈が混ざる                │
│   ・優先順位が曖昧になる        │
│   ・プロンプトが肥大化する      │
│   ・メンテが難しくなる          │
└──────────────────────────────┘
```

この時期に自然と気づいた。

> **“単一責任の原則は、エージェントにも効く”**

---

### ③ 次のエージェントを作るべきか？という葛藤

「じゃあ次のエージェントを作ろう」と思う一方で、

- 本当に必要？  
- 役割が重複しない？  
- 1つにまとめた方がいい？  

という迷いもあった。

---

## 🗂️ Vault構造が“効き始めた”のはこの頃

Intent → Context → Agents → Result → Daily

```
Intent   →   Context   →   Agents   →   Result   →   Daily
  🎯            📚            🤖            📦            📝
```

特に、

> **「このタスクは Intent として独立しているか？」**

という問いが、  
新しいエージェントを作る基準になった。

---

## 🔍 次に必要なエージェントが“自然と見えてきた”

1つ目のエージェントを使い込むうちに、  
次に必要な役割が浮かび上がってきた。

```
┌──────────────────────────────┐
│   🐣 1つ目のエージェントを使い込む │
└───────────────┬──────────────┘
                │
                ▼
        「あれ？ここ手動だな…」
                │
                ▼
┌──────────────────────────────┐
│   🔍 次に必要な役割が浮かび上がる │
│   ・スコープ生成                  │
│   ・TestRail出力                  │
│   ・日次ノート                    │
│   ・仕様変更検知                  │
└──────────────────────────────┘
```

まだ作ってはいない。  
でも、**必要性が見えてきた**。

---

## ✨ この時期の学び

- エージェントは“作って終わり”ではない  
- 使うことで次の課題が見える  
- 単一責任の原則はエージェントにも有効  
- Vault構造があると迷わない  
- 1つ目のエージェントが“次のエージェントのヒント”になる  

---

**1体 → 2体 → 3体**
