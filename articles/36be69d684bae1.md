# #43 Code Reviewer の内部構造（実装編）  
**― “PreToolUse Hook” がネットワーク全体の安全性をどう支えたか ―** 🛡️

Quality Reviewer が“基準点”なら、  
Code Reviewer は **“防波堤”** だった。

- 危険なコード変更を止める  
- diff を解析して安全性を判断する  
- PreToolUse Hook で全エージェントの操作を監視する  
- Quality Reviewer とは違う軸で品質を担保する  

このエージェントが存在したからこそ、  
ネットワークは **壊れずに進化** できた。

しかし、最初からこの形だったわけではない。  
Code Reviewer は **4回の構造刷新** を経て完成している。

---

# 🧩 1. 初期の問題：危険な操作を止められなかった

初期のネットワークでは、  
エージェントが自由にファイルを書き換えられた。

その結果：

### ■ 問題1：誤ったファイル編集が発生  
- ログファイルを上書き  
- 設定ファイルを破壊  
- JSON を壊す

### ■ 問題2：差分の安全性を判断できない  
「この変更は安全か？」を判断する仕組みがなかった。

### ■ 問題3：エージェント同士の干渉が起きる  
あるエージェントが書いた内容を、別のエージェントが壊す。

### ■ 問題4：ネットワーク全体の安定性が揺れる  
1つの誤操作が全体に波及した。

---

# 🧭 2. 方向性が決まった瞬間  
問題が積み上がった結果、  
Code Reviewer の役割は明確になった。

> **「すべてのコード変更を“通過点”として検査する」**  
> **「危険な操作は即座に止める」**  
> **「安全性の基準点になる」**

つまり、  
**“レビューするエージェント” ではなく “守るエージェント”** にする必要があった。

```
1. diff_parser（差分解析）
2. risk_analyzer（リスク判定）
3. pretooluse_hook（実行前フック）
```

この3層構造が揃って初めて、  
Code Reviewer は“防波堤”として機能した。

# 🔍 5. diff_parser（差分解析）

**背景：差分を正しく読めないと、リスク判定ができない。**

```python
def parse_diff(diff_text):
    changes = []
    for line in diff_text.split("\n"):
        if line.startswith("+") and not line.startswith("+++"):
            changes.append({"type": "add", "content": line[1:]})
        elif line.startswith("-") and not line.startswith("---"):
            changes.append({"type": "remove", "content": line[1:]})
    return changes
```

ポイント：

- “+++” や “---” は diff のメタ情報なので除外  
- 実際の追加・削除だけを抽出  
- 後続の risk_analyzer が扱いやすい形式に変換  

---

# ⚠️ 6. risk_analyzer（リスク判定）

**背景：危険な変更を検出するための“基準”が必要だった。**

```python
def analyze_risk(changes):
    risk = 0
    for c in changes:
        text = c["content"].lower()

        if "delete" in text or "rm -rf" in text:
            risk += 50

        if "password" in text or "token" in text:
            risk += 30

        if "config" in text and c["type"] == "remove":
            risk += 20

    return min(risk, 100)
```

検出していたのは：

- 危険コマンド  
- セキュリティ情報  
- 設定ファイルの削除  
- 重要ファイルの破壊

# 🛑 7. pretooluse_hook（実行前フック）

**背景：危険な操作を“実行前に止める”必要があった。**

```python
def pretooluse_hook(operation):
    diff = operation.get("diff", "")
    changes = parse_diff(diff)
    risk = analyze_risk(changes)

    if risk >= 50:
        return {
            "allowed": False,
            "reason": "High-risk change detected",
            "risk": risk,
            "changes": changes
        }

    return {"allowed": True, "risk": risk}
```

この Hook が導入されたことで、  
**ネットワーク全体の安全性が劇的に向上した。**
