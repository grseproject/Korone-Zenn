# #44 Spec Manager の内部構造（実装編）  
**― “真実の場所”をどう維持し、ネットワーク全体の判断を安定させたか ―** 📘

Spec Manager は、  
あなたのエージェントネットワークの中で **最も重要な“情報の基準点”** だった。

- Wiki（仕様）を読み  
- 差分を検出し  
- 自動更新し  
- Git と同期し  
- Analyzer に情報を渡し  
- Scope Generator の基準点になる  

つまり、  
**「仕様が揺れない世界」を作るためのエージェント**。

しかし、最初からこの形だったわけではない。  
Spec Manager は **5回の構造刷新** を経て完成している。

---

# 🧩 1. 初期の問題：仕様が揺れすぎた

初期のネットワークでは、  
仕様（Wiki）が揺れ続けていた。

### ■ 問題1：Wiki の更新漏れ  
- チケット更新  
- 実装変更  
- テスト追加  
これらが Wiki に反映されない。

### ■ 問題2：Scope Generator が古い仕様を参照  
結果として、  
**古い仕様でテストが生成される事故**が発生。

### ■ 問題3：Analyzer の判断がズレる  
仕様が揺れると、  
KPI や Issue triage の判断も揺れる。

### ■ 問題4：Git と Wiki の内容が一致しない  
“どれが正しい仕様なのか” が分からない状態に。

---

# 🧭 2. 方向性が決まった瞬間  
問題が積み上がった結果、  
Spec Manager の役割は明確になった。

> **「Wiki を“真実の場所”にする」**  
> **「差分を自動で検出し、揺れを止める」**  
> **「ネットワーク全体の基準点になる」**

つまり、  
**“仕様を書くエージェント” ではなく “仕様を守るエージェント”** にする必要があった。

---

# 🧱 3. 最終構造（4モジュール構成）

```
1. fetch_wiki（Wiki取得）
2. detect_diff（差分検出）
3. update_wiki（自動更新）
4. sync_git（Git同期）
```

この4つが揃って初めて、  
Spec Manager は“真実の場所”を維持できるようになった。

---


**1スキル＝1責務** に分割したことで、  
揺れがほぼゼロになった。

# 🔍 5. fetch_wiki（Wiki取得）

**背景：Wiki の構造が揺れるため、まず標準化が必要だった。**

```python
def fetch_wiki(page):
    raw = wiki_api.get(page)
    return parse_yaml(raw)
```

ポイント：

- YAML 形式に統一  
- Analyzer と Scope Generator が読みやすい形式に変換  
- Wiki の揺れを吸収する  

---

# 🔎 6. detect_diff（差分検出）

**背景：差分検出が不安定だと、更新漏れが発生する。**

```python
def detect_diff(local, wiki):
    diff = []
    for key in local:
        if local[key] != wiki.get(key):
            diff.append({
                "field": key,
                "local": local[key],
                "wiki": wiki.get(key)
            })
    return diff
```

検出していた差分：

- timeout  
- retry  
- API仕様  
- UI要素  
- エラーメッセージ  
- パラメータ  

**差分が1つでもあれば、更新対象。**

---

# 🔧 7. update_wiki（自動更新）

**背景：人手で更新すると必ず漏れる。**

```python
def update_wiki(page, diff):
    updated = apply_diff(page, diff)
    wiki_api.update(page, updated)
    return updated
```

自動更新の条件：

- 差分が軽微（例：数値変更）  
- 文言修正  
- UIラベル変更  

レビューが必要な場合は Analyzer に渡す。

---

# 🔄 8. sync_git（Git同期）

**背景：Wiki と Git がズレると、どちらが正しいか分からなくなる。**

```python
def sync_git(spec):
    path = f"specs/{spec['id']}.yaml"
    save_yaml(path, spec)
    git.commit(path, message="Sync spec with Wiki")
```

これにより：

- Wiki → Git  
- Git → Wiki  

の双方向同期が可能に。
