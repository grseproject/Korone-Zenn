#51 ガバナンス確立フェーズの入口 — ネットワークが“自分を統治し始めた”

14エージェントのネットワークは  
「壊れずに動く」「高速に回る」「品質を担保する」  
という 機能的完成 に到達した。

しかし3月に入ると、ネットワークは次の段階に踏み出す。

> “自分で自分を統治する” ための仕組みが生まれた。

ここからは、3月の更新で導入された  
ガバナンス層（Governance Layer） が  
ネットワーク全体をどう変えたのかを見ていく。

---

■ 1. ガバナンス層の誕生

3月下旬、4つの文書が追加された。

`
governance/
 ├─ agentcreationpolicy.md
 ├─ handoff_protocol.md
 ├─ behavioral_constraints.md
 └─ vaultstructurepolicy.md
`

これらは単なるルールではなく、  
ネットワークの“憲法” に近い。

● 役割の違い

| 文書 | 役割 |
|------|------|
| Agent Creation Policy | 新エージェントの誕生条件を定義 |
| Handoff Protocol | データ受け渡しの正規ルートを定義 |
| Behavioral Constraints | オーケストレーターの自己矯正ルール |
| Vault Structure Policy | ディレクトリ構造と命名規則の統一 |

これにより、ネットワークは  
「暗黙知で動く集合体」→「明文化された組織」  
へと変わった。

---

■ 2. Behavioral Constraints — 自己矯正の仕組み

特に重要なのが behavioral_constraints.md。

オーケストレーター自身の癖を9つに分類し、  
それぞれに対して 是正ルール を定義した。

例：

`
T-3: 委譲せず自分で実行してしまう
対策: 停止して委譲。オーケストレーターはディスパッチャーであり実行者ではない。
`

これは、ネットワークが初めて  
「自分の行動を自分で修正する」  
という能力を持った瞬間だった。

---

■ 3. Execution Gate — 実行権限の分離

ガバナンス層の導入と同時に、  
Execution Gate が明文化された。

`
オーケストレーターは Edit/Write/外部連携/CLI を直接実行してはならない。
`

これにより、  
- 実行者（ワーカー）  
- 判断者（オーケストレーター）  
- 助言者（Parallel Advisor）  

が明確に分離された。

● 擬似コード（Before → After）

Before（2月まで）

`pseudo
if task.requires_edit:
    orchestrator.run_edit()
`

After（3月以降）

`pseudo
if task.requires_edit:
    orchestrator.dispatch_to(worker)
    worker.requestprereview()
    if approved:
        worker.execute()
`

この構造は、後述する Pre-Review Model と密接に連動する。

---

■ 4. Knowledge Triage Router — 情報の秩序化

3月のもう一つの大きな変化がこれ。

`
情報を受け取る
  ↓
7段階のトリアージ
  ↓
正しい保存先へ自動振り分け
`

● ルーティング例（擬似コード）

`pseudo
if info.is_rule():
    save_to("rule/")
elif info.isagentspecific():
    save_to(f"agents/{agent}/config/")
elif info.istoolworkflow():
    save_to("tools/")
elif info.isproductspec():
    save_to("spec/")
elif info.is_shared():
    save_to("shared/")
elif info.is_snapshot():
    discard()
else:
    ask_user()
`

これにより、  
“とりあえず knowledge/ に保存” という混乱が消えた。

---

■ 5. Concierge Agent — 組織の記憶を管理する存在

Session Manager が進化し、  
Concierge Agent が誕生した。

concierge/
 ├─ daily_notes/
 ├─ activity/
 ├─ vault_health/
 └─ backup/

● 新しい役割

- 日次ノートの生成  
- 活動ログの記録  
- Vault の健全性チェック  
- MCP 経由の操作  
- イベント駆動バックアップ  

ネットワークは初めて  
「自分の歴史を自分で管理する」  
という能力を持った。
