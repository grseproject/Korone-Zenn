# #47 Session Manager の内部構造（実装編）  
**― “壊れないネットワーク”をどう実現したか ―** 🧩

Session Manager は、エージェントネットワークの中で **最も静かで、最も重要なエージェント** だった。

- 30分ごとの自動バックアップ  
- セッションの完全復旧  
- エージェント間の整合性維持  
- Orchestrator の状態管理  
- クラッシュ時の安全ネット  

これらを担うことで、  
**ネットワークが壊れずに進化できる基盤** を作っていた。

しかし、最初からこの形だったわけではない。  
Session Manager は **4回の構造刷新** を経て完成している。

---

# 🧩 1. 初期の問題：ネットワークが“壊れやすかった”

初期のネットワークでは、  
セッションが壊れると **全エージェントが巻き込まれて停止** していた。

### ■ 問題1：長時間の作業が突然消える  
- Scope Generator の途中  
- Analyzer の集計中  
- Reviewer のスコアリング中  

クラッシュすると **すべて消える**。

### ■ 問題2：Orchestrator の状態が壊れる  
どのタスクが終わっていて、  
どれが pending なのか分からなくなる。

### ■ 問題3：エージェント間の整合性が崩れる  
Spec Manager と Analyzer の情報がズレる。

### ■ 問題4：復旧が手動で、時間がかかる  
人間がログを見て復旧する必要があった。

---

# 🧭 2. 方向性が決まった瞬間  
問題が積み上がった結果、  
Session Manager の役割は明確になった。

> **「ネットワークを壊れなくする」**  
> **「クラッシュしても復旧できる構造を作る」**  
> **「エージェント間の整合性を維持する」**

つまり、  
**“安全ネット” としての役割** が必要だった。

---

# 🧱 3. 最終構造（3モジュール構成）

Session Manager は最終的に **3モジュール構成** に落ち着いた。

```
1. backup_state（状態のバックアップ）
2. restore_state（復旧）
3. validate_integrity（整合性チェック）
```

構成図はこう。

```
+---------------------------+
|      Session Manager      |
|---------------------------|
| backup_state              |
| restore_state             |
| validate_integrity        |
+-------------+-------------+
              |
              v
        [Orchestrator]
```

# 🔍 5. backup_state（30分ごとのバックアップ）

**背景：長時間の作業が消える事故が多かった。**

```python
def backup_state(state):
    timestamp = now()
    path = f"backup/session_{timestamp}.json"
    save_json(path, state)
    return path
```

バックアップ対象：

- Orchestrator のタスク状態  
- 各エージェントの内部キャッシュ  
- 最新の Wiki/Spec 情報  
- Analyzer の中間結果  
- Reviewer のスコア履歴  

**30分ごとに自動実行。**

---

# 🔧 6. restore_state（復旧）

**背景：クラッシュ後の復旧が手動で時間がかかっていた。**

```python
def restore_state(path):
    state = load_json(path)
    return state
```

復旧の流れ：

1. 最新バックアップを読み込む  
2. Orchestrator の pending タスクを復元  
3. Analyzer の中間結果を復元  
4. Spec Manager の同期状態を復元  
5. Reviewer のスコア履歴を復元  

**復旧は数秒で完了。**

---

# 🧪 7. validate_integrity（整合性チェック）

**背景：エージェント間の情報ズレが事故の原因だった。**

```python
def validate_integrity(state):
    issues = []

    if state["spec"]["version"] != state["wiki"]["version"]:
        issues.append("Spec/Wiki version mismatch")

    if state["orchestrator"]["pending"] < 0:
        issues.append("Invalid pending count")

    return issues
```

チェックしていた整合性：

- Spec Manager ↔ Wiki  
- Analyzer ↔ Spec  
- Orchestrator ↔ Reviewer  
- タスクの pending 数  
- JSON の破損
